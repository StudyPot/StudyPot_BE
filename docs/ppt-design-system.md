# StudyPot — 경진대회 발표 PPT 디자인 시스템 & 제작 브리프

> 이 문서를 **「AI Study Leader 기능 명세서 v3」와 함께** Claude Code에 넘기면, 일관된 브랜드 PPTX가 나옵니다.
> 목적: 관통 PJT 경진대회 심사 발표 · 출력: `.pptx`(16:9) · 비주얼: 제품 브랜드(그린+라이트)

---

## 0. Claude Code에게 주는 지시 (이 블록을 그대로 프롬프트에 포함)

```
첨부한 두 문서로 발표용 PPTX를 만들어줘.
- 내용 원본: 「AI Study Leader 기능 명세서 v3」 (제품 기획/기능)
- 디자인 규칙: 이 「PPT 디자인 시스템」 문서 (색/폰트/레이아웃/슬라이드 템플릿)

요구사항:
1. anthropic-skills:pptx 스킬을 사용해 python-pptx로 .pptx를 생성한다.
2. 화면비 16:9 (13.333in × 7.5in). 아래 디자인 토큰(색 HEX/RGB, 폰트, pt 크기)을 정확히 지킨다.
3. §6 슬라이드 아웃라인(15장)을 그대로 슬라이드로 만든다. 한 슬라이드에 글머리 6줄 이내.
4. 표·KPI 카드·칩(배지)·플로우 화살표는 §5 컴포넌트 레시피대로 그린다.
5. 모든 슬라이드 하단에 푸터(좌: "StudyPot · AI Study Leader", 우: 페이지 번호).
6. 한국어 본문. 폰트는 Pretendard(없으면 맑은 고딕) 지정.
7. 완성 후 슬라이드 수·각 장 제목을 요약 보고.
```

## 1. 브랜드 기반 (Brand Foundation)

- **컨셉**: 뱅크샐러드풍 "신뢰감 있는 핀테크/툴" 톤. 밝은 화이트 배경 + 생기 있는 그린 포인트. 깔끔·정돈·여백.
- **인상 키워드**: 똑똑한 / 자동화된 / 믿음직한 / 가벼운
- **금지**: 무지개색 남용, 그라데이션 과다, 꽉 찬 텍스트, 저대비 회색 본문, 클립아트.

## 3. 타이포그래피

- **폰트**: 본문/제목 모두 **Pretendard** (없으면 `맑은 고딕`/`Apple SD Gothic Neo`). 숫자 강조엔 동일 폰트 Bold.
- python-pptx에서 `run.font.name = "Pretendard"` 지정. (Pretendard 미설치 환경 대비 맑은 고딕 fallback 주석)

### 스케일 (16:9 기준, pt)
| 용도 | 크기 | 굵기 | 색 |
|---|---|---|---|
| 표지 타이틀 | 40–44 | Bold | Ink / Primary Deep |
| 섹션 디바이더 | 40 | Bold | 흰색(그린 배경) |
| 슬라이드 제목 | 28–32 | Bold | Ink |
| 소제목/리드 | 20 | SemiBold | Primary Deep |
| 본문 | 18 | Regular | Ink |
| 글머리 보조 | 16 | Regular | Text Secondary |
| KPI 큰 숫자 | 60–72 | Bold | Primary Deep |
| 캡션/푸터 | 12 | Regular | Muted |
| 칩(배지) | 12 | SemiBold | 흰색 또는 Primary Deep |

- 줄간격 1.3, 한글 잘림 방지(타이트한 line-height 금지).

## 4. 레이아웃 그리드

- 캔버스 13.333in × 7.5in. **여백(safe margin) 0.6in** 사방.
- 가로 12컬럼 가정(컬럼폭 ≈ 0.95in, gutter 0.2in). 2단/3단 카드 정렬에 사용.
- **제목 영역**: 상단 0.6in~1.6in. 제목 좌측 정렬, 제목 아래 그린 짧은 언더라인(폭 0.8in, 두께 4pt, `#19C37D`).
- **본문 영역**: 1.8in~6.7in.
- **푸터**: y≈7.0in. 좌측 워드마크 텍스트 "StudyPot · AI Study Leader"(Muted 12pt), 우측 페이지 번호.
- 정렬 일관성: 모든 카드/표는 같은 좌측 기준선. 요소 간 최소 간격 0.2in.

## 5. 컴포넌트 레시피

### 5.1 칩 / 배지 (도메인·기능 라벨용)
- 둥근 사각형(rounded rect), 높이 0.28in, 좌우 패딩 0.12in, 폰트 12pt SemiBold.
- 종류: **신규**=채움 `#19C37D`+흰 글자 / **재설계**=채움 `#0E8F5A`+흰 글자 / **보류**=채움 `#F5A623`+흰 글자 / **기존**=테두리 `#E5E7EB`+Secondary 글자(배경 흰색).

### 5.2 KPI 카드 (예: 28→82)
- 카드: 흰 배경, 1pt `#E5E7EB` 테두리 또는 `#F4FBF8` 채움, 모서리 8px.
- 구성: 큰 숫자(60–72pt, Primary Deep) + 라벨(14pt Secondary). 변화는 "28 → 82" 한 줄, 화살표 그린.
- 3~4개를 가로로 등간격 배치.

### 5.3 콜아웃 박스 (핵심 메시지/인용)
- 좌측 4pt 그린 바 + `#E6F9F1` 배경 + Ink 18pt. 슬라이드당 1개만.

### 5.4 플로우(골든패스/아키텍처)
- 둥근 사각형 노드(흰 배경, 그린 테두리 1.5pt, 제목 16pt Bold) + 그린 화살표(`#19C37D`, 2pt).
- 단계 노드는 좌→우 또는 지그재그. 노드 안 텍스트 2줄 이내.
- 아키텍처: 레이어 박스(FE / API / Domain / DB·LLM·SSE)를 위→아래 stack, 레이어명은 Primary Tint 헤더.

### 5.5 표 스타일
- 헤더 행: 채움 `#D1F2E3`, 글자 Primary Deep 14pt Bold.
- 바디 행: 흰/`#F4FBF8` 교차(zebra), 글자 Ink 14pt, 1pt `#E5E7EB` 가로선만(세로선 최소화).
- "신규" 등 상태 컬럼은 칩으로 표시.

### 5.6 아이콘
- 라인 아이콘(단색 `#0E8F5A`), 굵기 균일. 도메인별 1개씩(그룹·커리큘럼·회고·리포트·알림·게시판 등). 이모지 대체 가능하나 톤 통일.

## 6. 슬라이드 아웃라인 (15장 · 경진대회 서사)

> 흐름: **문제 → 솔루션 → 동작 → 기술 → 성과/규모 → 로드맵 → 마무리**. 각 장 (템플릿) 표기.

1. **표지** (Cover) — "StudyPot — AI가 스터디 팀장을 대신한다" + 한 줄 정의 + 팀/날짜. 그린 포인트.
2. **문제 정의** (콜아웃+3불릿) — 소규모 스터디의 운영 공백: 사람 매니저 부재, 진행 추적·피드백 어려움. (v3 §1 타깃)
3. **솔루션 한 줄** (대형 콜아웃) — "호스트는 그룹만, 나머지는 AI 팀장이 운영" + 6 핵심가치 칩. (v3 §1)
4. **MVP 골든패스** (플로우 5.4) — 생성→초대→온보딩→시작→AI 커리큘럼→TODO/회고→자동 리포트. (v3 §2)
5. **데모 하이라이트** (2단 카드) — 화면 캡처 자리표시 + 한 줄 설명 3~4개(그룹홈·TODO·회고·AI 대화). *스크린샷 후입력*
6. **시스템 아키텍처** (레이어 다이어그램 5.4) — FE(Vue FSD) / REST `/api/v1` / 15 도메인 / MySQL8·SSE·LLM. (v3 §3)
7. **도메인 맵** (표 5.5) — 15개 도메인 + 역할 + 신규 칩. (v3 §4)
8. **핵심 기능 ① 커리큘럼·주차 TODO** (2단 불릿+아이콘) — AI 커리큘럼 생성, 활동 잔디/히트맵/피드. (v3 §5.4)
9. **핵심 기능 ② 회고 + 자동 리포트** (플로우+콜아웃) — 리커트+서술 설문, 주차 잠금, 주차 마감 시 AI 팀장 리포트 자동 게시. (v3 §5.5–5.6)
10. **핵심 기능 ③ AI 팀장 & 실시간** (2단) — 퍼소나, 대화 SSE 스트림, 인앱 알림 16종 SSE. (v3 §5.7, §5.11)
11. **협업 기능** (3단 카드) — 게시판/댓글, 그룹 규칙·위반, 리뷰·북마크·팔로우. (v3 §5.8–5.10)
12. **기술 포인트** (불릿+칩) — Java21/Spring·JDBC, UUIDv7 BINARY(16), BFF+HttpOnly+CSRF, RFC9457, Flyway V1→V10, 851 테스트. (v3 §3,§7)
13. **규모/성과** (KPI 카드 5.2) — 엔드포인트 28→82, 도메인 8→15, enum 16→28, 마이그레이션 10, 테스트 851통과. (v3 §6,§8)
14. **로드맵** (타임라인+앰버 칩) — 커리큘럼 점진 생성(#18 A~D), 리뷰 모델 통합 결정. (v3 §10)
15. **마무리/Thank you** (콜아웃) — 한 줄 비전 + 링크/QR 자리.

> 분량 조절: 8~11번을 줄이면 12장, 데모를 늘리면 17장까지. 기본 15장 권장.

## 7. 품질 체크리스트 (생성 후 자가 검수)

- [ ] 모든 텍스트 색 대비 충분(그린 위 흰색 / 라이트 위 Ink). Muted는 캡션만.
- [ ] 한 슬라이드 글머리 6줄 이내, 폰트 16pt 미만 없음(표/캡션 제외).
- [ ] 색은 §2 팔레트 안에서만. 임의 색 금지.
- [ ] 제목 좌측 정렬 + 그린 언더라인 일관.
- [ ] 푸터(워드마크+페이지번호) 전 슬라이드 동일 위치.
- [ ] 표지/디바이더 외 흰 배경 유지, 그린은 포인트로만(면적 20% 이내).
- [ ] 폰트 Pretendard 지정(미설치 대비 맑은 고딕 fallback 주석).
- [ ] 숫자/KPI는 Bold + Primary Deep로 시선 집중.
