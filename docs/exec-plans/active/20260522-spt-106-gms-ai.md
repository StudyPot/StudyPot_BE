## Goal
Register the GMS AI key as a GitHub Secret, deploy StudyPot so production AI calls use the GMS OpenAI-compatible `chat/completions` endpoint, and verify both provider-level and StudyPot-level real AI calls without exposing the secret.

## Approach
Keep the existing OpenAI Responses API behavior as the default so current OpenAI-compatible deployments do not regress. Add a small configurable provider mode for `responses` vs `chat-completions`, selected by a new runtime env var. In `chat-completions` mode, map `LlmStructuredRequest` to a developer/user message pair, translate the existing JSON schema format into Chat Completions `response_format`, and parse `choices[0].message.content` plus `usage.prompt_tokens` / `usage.completion_tokens`.

Use GitHub Secrets for the GMS key, base URL, model, and API mode. Update the Deploy workflow to upload an Actions-managed runtime env file to the deploy directory and include it in `docker compose --env-file` calls, so future redeploys do not require direct server edits for the AI key. Keep the secret value out of docs, logs, commits, and PR text.
